# Report #1: Happened 기획서와 디자인 방향

## 이번 보고 요약

Happened의 MVP 기준 문서와 주요 앱 화면 디자인 보드를 만들었다. 현재 방향은 "장소에 도착해야 완전히 열리는 추억 피드 앱"이며, 홈 화면은 지도보다 먼저 보이는 풀스크린 세로 피드로 잡았다.

## 산출물

- MVP 기획서: `docs/product-brief.md`
- 제품 규칙: `docs/product-rules.md`
- 디자인 프로토타입 설명: `docs/design-prototype.md`
- 주요 앱 화면 디자인 보드: `docs/prototypes/happened-major-screens.svg`
- 보고 채널 검토: `docs/reporting-channels.md`

## 디자인 보드 포함 화면

- 온보딩/인증
- 홈 열린 기억
- 홈 잠긴 프리뷰
- 지도
- 촬영/체크인
- 장소 상세
- 재방문 회상
- 프로필/안전 설정

## 판단

SSH 기반 맥미니 환경에서는 GitHub 저장소 안에 `reports/` 디렉터리를 두고 `Report #1`, `Report #2`처럼 누적하는 방식이 가장 단순하다. Discord보다 설정이 적고, 문서/이미지/코드 변경이 같은 히스토리에 남는다.
