# Happened MVP 기획서

## 한 줄 정의

Happened는 장소에 남긴 순간이 그 장소에 다시 도착했을 때 완전히 열리는 위치 잠금형 추억 앱이다.

## 제품 가설

대부분의 SNS는 사람, 시간, 관심사 알고리즘을 기준으로 기억을 정리한다. Happened는 장소를 기준으로 기억을 정리한다. 카페, 학교, 회사, 공연장, 골목, 역 같은 공간이 이야기를 담는 저장소가 되고, 사용자는 위치 검증을 거쳐 그 장소에 콘텐츠를 남기며, 다시 가까이 갔을 때 완전히 열람한다.

첫 화면은 지도가 아니다. 첫 화면은 몰입형 세로 미디어 피드다. 다만 모든 게시물은 일반 SNS 콘텐츠보다 먼저 "어디에서 있었던 일인가"가 읽혀야 한다.

## 타깃 사용자

- 개인 추억 사용자: 의미 있는 장소에 다시 갔을 때 예전 기억이 자연스럽게 떠오르길 원한다.
- 친구/장소 공유 사용자: 학교, 회사, 카페, 동네 같은 장소에 친구들의 에피소드가 쌓이는 경험을 원한다.
- 크리에이터/팬덤 사용자: 유명인이 남긴 장소 기반 콘텐츠를 직접 방문해서 열람하고 싶어 한다. 이 시나리오는 v2 확장 후보로 둔다.

## 핵심 약속

- 여기서 추억을 남긴다.
- 다시 여기 오면 열린다.
- 친구들이 남긴 장소의 이야기를 실제로 방문하며 경험한다.

## V1 제품 형태

- iOS/Android 중심 모바일 앱.
- 첫 화면은 풀스크린 세로 피드.
- 장소명, 거리, 잠금 상태, 공개 범위가 좋아요나 프로필 장식보다 먼저 보이는 정보 구조.
- 위치는 상시 추적이 아니라 체크인/해제 시점의 이벤트성 검증 중심.
- 기본 공개 범위는 팔로워 전용.
- 백엔드와 인프라 결정 전까지는 더미 데이터 기반으로 UX 완성도를 먼저 끌어올린다.

## 핵심 루프

1. 발견: 앱을 열고 Following, Nearby, Memories 피드를 본다.
2. 해제: 장소 근처에 접근하면 블러/잠금 콘텐츠가 열린다.
3. 기록: 장소 근처에서 체크인하고 사진/영상을 남긴다.
4. 재방문: 예전에 머문 장소에 다시 가면 회상 카드나 알림이 뜬다.
5. 축적: 장소 상세에서 월별로 쌓인 이야기를 본다.

## V1 기능 범위

- 이메일 회원가입/로그인.
- Google 로그인.
- Apple 로그인.
- 위치, 카메라, 사진, 알림 권한 온보딩.
- 현재 위치 기반 장소 선택.
- 12시간 유효한 체크인 토큰.
- 체크인 이후 사진/영상 업로드.
- 팔로우/팔로워.
- Following, Nearby, Memories 피드.
- 장소 근처에서만 완전 열람.
- 반경 밖에서는 블러 프리뷰.
- 장소 상세와 월별 타임라인.
- 히트맵 스타일 지도와 장소 버블, 내 추억 핀.
- 재방문 회상 카드와 알림 표면.
- 신고, 차단, 숨김, 계정 삭제.

## 위치 규칙

- 업로드 가능 조건: 선택 장소 120m 이내, 위치 정확도 50m 이하.
- 나중 업로드 허용: 체크인 성공 후 12시간.
- 완전 열람 해제: 장소 200m 이내.
- 대형 장소: 데이터 구조상 최대 300m까지 확장 가능.
- 민감한 사적 장소: 공개 게시 제한 또는 위치 정밀도 약화.

## 초기 정보 구조

- Home: 장소 잠금 상태가 강조되는 몰입형 피드.
- Map: 히트맵, 장소 버블, 내 추억 핀.
- Capture: 검증된 장소 촬영과 12시간 체크인 토큰.
- Timeline: 내 추억과 장소별 기억의 시간 축.
- Profile: 신원, 팔로우, 공개 범위, 안전 설정.

## 디자인 방향

- 차콜 네이비 기반의 어두운 화면.
- 시안, 라임, 코랄, 옐로, 레드를 상태와 장소 에너지의 포인트로 사용.
- 짧은 영상 앱처럼 몰입감 있는 홈.
- 지도 서비스처럼 딱딱하지 않은 공간감.
- 필름 다이어리 감성은 유지하되 과하게 감상적으로 가지 않는다.
- 컨트롤이 필요한 곳에만 매트한 반투명 패널을 쓴다.

## MVP 성공 기준

- 사용자가 첫 화면에서 콘텐츠가 장소에 묶여 있다는 점을 바로 이해한다.
- 잠김/열림 상태가 2초 안에 읽힌다.
- 홈 화면이 임시 화면이 아니라 실제 제품처럼 느껴진다.
- 촬영 플로우에서 업로드는 현장 검증이 필요하다는 점이 명확하다.
- 장소 상세가 단순 목록이 아니라 기억 아카이브처럼 느껴진다.
- 실제 지인 테스트를 시작할 수 있을 만큼 신고/차단/삭제 진입점이 보인다.

## 근시일 빌드 계획

1. 디자인 프로토타입과 앱 셸을 정리한다.
2. Home, Map, Capture, Timeline, Profile, Place Detail, Auth, Permission Onboarding, Recall을 더미 데이터로 고도화한다.
3. 잠금/해제, 체크인 토큰, 공개 범위, 신고/차단, 빈 상태, 에러 상태의 앱 상태 모델을 만든다.
4. 백엔드, DB, 파일 저장소, 인증 제공자, 푸시, 테스트앱 배포 방식은 인프라 승인 시점에 결정한다.
