# Happened 디자인 프로토타입

## 프로토타입 파일

정적 디자인 보드:

[docs/prototypes/happened-major-screens.svg](prototypes/happened-major-screens.svg)

이 보드는 디자인 확인용이다. 백엔드, 실제 미디어, 네이티브 지도, 인증 제공자, 배포 방식 결정 전에도 완성 앱이 어떤 방향으로 보일지 빠르게 판단하기 위한 산출물이다.

## 포함 화면

- Onboarding/Auth: 제품 약속과 권한 요청 흐름.
- Home/Open Memory: 장소명, 거리, 잠금 상태가 최우선으로 읽히는 풀스크린 피드.
- Home/Locked Preview: 해제 반경 밖 콘텐츠의 블러 프리뷰 상태.
- Map: 히트맵 스타일 장소 필드, 장소 버블, 핀, 잠금/열림 상태.
- Capture: 장소 검증, 체크인 토큰, 12시간 나중 업로드 윈도우.
- Place Detail: 특정 장소에 월별로 쌓이는 이야기.
- Recall: 재방문 시 예전 기억이 다시 열리는 카드.
- Profile/Safety: 팔로워 기본 공개, 신고/차단, 알림, 계정 삭제 진입점.

## 디자인 결정

- 홈 화면을 가장 강한 첫인상으로 둔다.
- 지도는 제품의 본질을 보조하지만 첫 경험을 지배하지 않는다.
- 모든 화면에서 장소 정보가 일반 소셜 지표보다 먼저 보인다.
- 잠긴 콘텐츠는 완전히 숨기지 않고, 맥락만 보여주며 현장 방문 동기를 만든다.
- 촬영 화면은 생성보다 먼저 검증 상태를 전달한다.
- 위치 기반 제품이므로 안전/프라이버시 진입점은 초기 화면군 안에 포함한다.

## 구현 메모

- React Native 앱은 이 보드의 방향성을 따르되 픽셀 단위 복제보다 실제 모바일 조작감을 우선한다.
- 실제 사진/영상은 이후 더미 그라데이션을 대체한다.
- 실제 지도 SDK 선택은 인프라, 비용, 배포 의사결정과 연결되므로 지금은 보류한다.
- 앞으로 추가되는 화면도 어둡고 몰입감 있는 장소 우선 언어를 유지한다.
- 새 디자인 피드백 기준으로 아날로그 필름 감성을 더한다. 필름 프레임, 그레인, 라이트 리크, 날짜 스탬프, 컨택트시트 리듬을 쓰되 장소 정보의 우선순위는 유지한다.
